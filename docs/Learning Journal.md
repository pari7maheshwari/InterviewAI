# Session 7

## What I Learned

- A Product Requirements Document (PRD) focuses on the product, not the implementation.
- Product decisions should always solve a user problem.
- Great onboarding demonstrates value before asking users to commit.
- User friction should be minimized by collecting information only when it improves the current experience.
- Product principles help maintain consistency throughout development.

## Session 8

### What I Learned

- User personas are useful, but user goals are even more important.
- People use products to accomplish a job, not to use a feature.
- Every onboarding question should have a clear purpose.
- The user's goal should influence the entire interview journey.

## Session 9

### What I Learned

- Edge cases are as important as the happy path.
- A good product guides users instead of rejecting them.
- Every failure should provide a recovery path.
